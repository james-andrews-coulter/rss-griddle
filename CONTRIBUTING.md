# Contributing

Contributions are welcome! Here's how to help:

## Reporting Issues

Open a GitHub issue with:
- What you expected
- What happened instead
- Steps to reproduce

## Submitting Changes

1. Fork the repo
2. Create a branch (`git checkout -b my-fix`)
3. Make your changes
4. Run tests: `go test -v ./...`
5. Commit and open a PR

## Code Style

- Follow existing patterns in `main.go`
- Run `go vet ./...` before submitting
- Keep it simple — this is a ~650-line app on purpose
